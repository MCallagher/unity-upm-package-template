# How to hide meta files in VSCode
Unity and C# generates many files that, despite being important, are not useful in the process of development but contribute to the visual disorder in the repository. Therefore, it can be useful to hide, without deleting, these files and folders to have a cleaner development environment. In VSCode this can be achieved in just a few steps.

## 1. Create a .vscode folder
If it does not already exists, create a `.vscode` folder in the root of the repository.

## 2. Create a settings.json file
If it does not already exists, create a `settings.json` file under the `.vscode` folder.

## 3. Specify the files and directory to ignore
In the `settings.json` file, add the list of patterns that you want to ignore (hide in the VSCode explorer).

This sample hides the `*.meta` files generated by Unity and the `obj` and `bin` folders generated by C#.

```
{
    "files.exclude": {
        "**/*.meta": true,
        "obj/": true,
        "bin/": true
    }
}
```

## Conclusion
This simple setting visually cleans the VSCode editor and improve the overall understanding of the repository components that are useful in the developing process. All without deleting any file.
